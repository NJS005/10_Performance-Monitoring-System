📌 Project Title
AMS: A Role-Based Performance Tracking & Academic Monitoring Portal

Current academic monitoring processes often rely on fragmented manual records or insecure spreadsheets, making it difficult for Faculty Advisors (FAs) to track student progress in real-time. Students lack a centralized dashboard to view their attendance, grades, and eligibility criteria (like the 80% attendance rule). This project aims to bridge that gap by providing a secure, automated platform for:

Centralized Data: Storing student performance, batch details, and course history in a structured PostgreSQL database.

Secure Access: Preventing unauthorized access through Google OAuth 2.0 and JWT-based role guarding.

Automated Monitoring: Enabling FAs to identify at-risk students based on academic thresholds.

💻 Technology Stack
Frontend
React.js: For a  component-based user interface.

Backend
Spring Boot (Java 25): Providing a robust RESTful API architecture.

Database
PostgreSQL 18: A relational database for storing student, course, and user information.
